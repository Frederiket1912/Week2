Explain (conceptually) about the TCP/IP-protocol stack
TCP/IP stacken beskriver hvordan man sender data på en maskine på et netværk, til en anden maskine på samme netværk (fx internettet).
Detaen starter i applicationen og bliver først sendt til transportlaget hvor dataen bliver segmenteret og hver segment får en header.
Det er i transportlaget at TCP protokollen bestemmer hvilke informationer headeren skal have.
Efter transportlaget bliver segmenterne sendt til netværkslaget hvor de bliver delt ud i packets og får en ny header.
I transportlaget er det IP protokollen der bestemmer hvilke informationer headeren skal have.
Så ryger packets ned til Data Link laget, hvor de bliver opdelt igen i frames og får en ny header beskrevet af ethernet protokollen.
Til sidst ryger disse frames fysisk ud i netværket enten i ledninger eller trådløst som bits. Denne strøm af bits har både en head og en tail,
så man ved hvormår transmitionen starter og slutter.


Explain about the HTTP Protocol (the following exercises will go much deeper into this protocol)
HTTP er en protokol der beskriver hvordan man kan hente ressourcer fx html dokumenter, css, billedfiler osv. Set i forhold til TCP/IP stacken
fungerer HTTP i appplicationslaget. Client (ofte en browser) og server kommunikere vha requests og responses, til forskel fra en strøm af data.
